  <action>
    The `localePrefix: "as-needed"` configuration means:
    - Spanish (default locale): URLs have NO prefix -> `https://platosvivaz.com/productos`
    - English: `/en/productos`
    - French: `/fr/productos`
    - German: `/de/productos`

    This creates a canonical/hreflang challenge: the Spanish version has a different URL pattern than the other 3 locales.

    1. In `frontend/src/app/[locale]/layout.tsx`, add hreflang and canonical `<link>` tags in the `<head>` section. Add this inside the `<html>` tag, before `<body>`:

       First, compute the alternateUrls in `generateMetadata`:
       ```typescript
       const siteUrl = process.env.NEXT_PUBLIC_SITE_URL || "https://platosvivaz.com";
       // For localePrefix: "as-needed", default locale (es) has no prefix
       const alternates = {
         canonical: locale === "es" ? siteUrl : `${siteUrl}/${locale}`,
         languages: {
           es: siteUrl,         // No prefix for default locale
           en: `${siteUrl}/en`,
           fr: `${siteUrl}/fr`,
           de: `${siteUrl}/de`,
           "x-default": siteUrl, // Default hreflang points to root (Spanish)
         },
       };
       ```

       Add `alternates` to the returned Metadata object. This is the Next.js Metadata API way to handle hreflang. The alternates should be path-aware — for inner pages like /productos, the alternates should include the path.

       Since the layout metadata applies globally, and inner pages have their own generateMetadata, the approach is:
       - Keep layout metadata for the root URL
       - Each page's generateMetadata should include its own alternates

       For now, ensure the layout generateMetadata includes alternates for the root URL. Page-level alternates are a Phase 5 polish item.

    2. Verify `frontend/src/app/sitemap.ts` includes all locale variants. Read the current sitemap.ts and ensure it generates URLs for all 4 locales with the correct prefix pattern (no prefix for es, /en/ for en, etc.).

       If the sitemap does not account for locale variants, update it to generate entries like:
       ```typescript
       const pages = ["", "productos", "tecnologia", "sobre-vivaz", "regulacion-2026", "contacto", "noticias"];
       const locales = ["es", "en", "fr", "de"];
       const siteUrl = process.env.NEXT_PUBLIC_SITE_URL || "https://platosvivaz.com";

       const urls = pages.flatMap(page =>
         locales.map(locale => ({
           url: locale === "es"
             ? `${siteUrl}/${page}`
             : `${siteUrl}/${locale}/${page}`,
           lastModified: new Date(),
           alternateRefs: locales.map(l => ({
             hreflang: l,
             href: l === "es"
               ? `${siteUrl}/${page}`
               : `${siteUrl}/${l}/${page}`,
           })),
         }))
       );
       ```

    3. Run `npm run build` to confirm the metadata changes compile.
