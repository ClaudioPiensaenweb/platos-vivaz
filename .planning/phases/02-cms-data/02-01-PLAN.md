Create a new file `backend/seed-pdf.js` that uploads the shared catalog PDF to Directus file storage and links the resulting UUID to the sys_brand singleton.

The script should:

1. Use the same boilerplate as seed-data.js: ESM imports, dotenv config from parent .env, createDirectus with staticToken.

2. Import `uploadFiles` and `updateSingleton` from `@directus/sdk`. Import `readFileSync` from `fs` and `resolve`, `dirname` from `path`.

3. Main function:
   ```javascript
   async function main() {
     console.log('============================================');
     console.log(' VIVAZ - Upload Catalog PDF');
     console.log('============================================');

     const pdfPath = resolve(__dirname, '..', 'Catalog-min.pdf');
     console.log('  Reading PDF from:', pdfPath);

     const pdfBuffer = readFileSync(pdfPath);
     const blob = new Blob([pdfBuffer], { type: 'application/pdf' });

     const formData = new FormData();
     formData.append('title', 'Vivaz Clay Targets - Product Catalog');
     formData.append('filename_download', 'vivaz-catalog.pdf');
     formData.append('file', blob, 'vivaz-catalog.pdf');

     try {
       const file = await client.request(uploadFiles(formData));
       console.log('  [OK] PDF uploaded, UUID:', file.id);

       // Link to sys_brand singleton
       await client.request(updateSingleton('sys_brand', {
         catalog_pdf: file.id,
       }));
       console.log('  [OK] sys_brand.catalog_pdf linked');
     } catch (e) {
       if (e.message?.includes('already exists')) {
         console.log('  [SKIP] PDF may already exist:', e.message?.substring(0, 80));
       } else {
         console.error('  [ERR] PDF upload:', e.message || e);
       }
     }
   }

   main();
   ```

4. Verify Catalog-min.pdf exists at the project root before writing this script. The file is listed in the git status as an untracked file at the project root.

NOTE: This uses Node 22 built-in FormData and Blob -- no extra packages needed. The backend runs on node:22-alpine per docker-compose.yml.
